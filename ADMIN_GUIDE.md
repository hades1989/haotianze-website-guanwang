# 管理后台使用指南

## 环境配置

### 1. 设置管理员账号密码

复制 `.env.local.example` 文件为 `.env.local`，并修改默认密码：

```bash
cp .env.local.example .env.local
```

然后编辑 `.env.local` 文件：

```env
ADMIN_USERNAME=your_username
ADMIN_PASSWORD=your_strong_password
```

**重要提示**：
- 请使用强密码
- 不要将 `.env.local` 文件提交到版本控制
- 默认密码仅在开发环境使用

### 2. 部署到生产环境

在生产环境中，建议使用环境变量管理工具（如 Vercel 环境变量）来配置：

- `ADMIN_USERNAME`
- `ADMIN_PASSWORD`

## 功能说明

### 1. 登录管理后台

- 访问路径：`/admin/login`
- 或点击网站页脚的"管理员登录"链接
- 输入用户名和密码登录

### 2. 管理后台功能

目前支持的管理功能：

#### 上传案例图片
- 路径：`/admin/upload`
- 功能：为成功案例上传展示图片
- 使用方法：选择案例 → 上传图片 → 获取路径 → 更新代码

### 3. 退出登录

- 点击管理后台右上角的"退出登录"按钮
- 会话将立即失效，需要重新登录才能访问管理功能

## 安全建议

1. **修改默认密码**：首次使用时立即修改管理员密码
2. **定期更换密码**：建议每 3-6 个月更换一次密码
3. **使用强密码**：密码应包含大小写字母、数字和特殊字符
4. **限制访问**：在生产环境中，可以限制管理后台的 IP 访问
5. **启用 HTTPS**：确保网站使用 HTTPS 协议

## 技术说明

### 认证机制

- 使用 Cookie-based Session 认证
- Session 有效期：24 小时
- Cookie 属性：
  - `httpOnly`: 防止 XSS 攻击
  - `secure`: 生产环境仅通过 HTTPS 传输
  - `sameSite`: 防止 CSRF 攻击

### 路由保护

- 使用 Next.js Middleware 保护 `/admin/*` 路由
- 未登录用户访问管理页面时自动重定向到登录页
- 登录页面除外（`/admin/login`）

### API 端点

| 端点 | 方法 | 描述 |
|------|------|------|
| `/api/auth/login` | POST | 用户登录 |
| `/api/auth/logout` | POST | 用户登出 |
| `/api/auth/check` | GET | 检查登录状态 |

## 扩展功能建议

如需进一步强化安全性，可以考虑：

1. **双因素认证（2FA）**
2. **登录失败次数限制**
3. **登录日志记录**
4. **IP 白名单**
5. **密码复杂度验证**
6. **自动登出（长时间无操作）**
7. **管理员权限分级**

## 故障排查

### 无法登录

- 检查用户名和密码是否正确
- 确认 `.env.local` 文件配置正确
- 清除浏览器 Cookie 后重试

### 管理后台无法访问

- 确认已成功登录
- 检查 Cookie 是否过期（24小时）
- 尝试重新登录

### 环境变量未生效

- 确认环境变量名称正确（`ADMIN_USERNAME`, `ADMIN_PASSWORD`）
- 重启开发服务器
- 检查 `.env.local` 文件是否在项目根目录
